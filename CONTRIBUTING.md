# Contributing

Before making a substantial change, open an issue or discuss the change with
the maintainers so the scope and compatibility impact are clear.

## Pull Request Process

1. Keep changes focused on one concern.
2. Preserve existing public method signatures unless a breaking change has been
   explicitly accepted.
3. Update documentation when behavior, installation, commands, or public APIs
   change.
4. Run the project checks before submitting:

```bash
make qa
```

## Code of Conduct

Contributors are expected to communicate respectfully, accept constructive
feedback, and focus on what is best for the project.

Unacceptable behavior includes harassment, personal attacks, publishing private
information without permission, and other conduct that would reasonably be
considered inappropriate in a professional setting.

Reports can be sent to the project maintainers.
